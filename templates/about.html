Analysis Parameters

Core dimensions of our deep-scan technology

report_problem

Syntax Errors

We check for Python syntax errors that prevent code execution. Invalid syntax is identified using Python’s AST parser.

Example: Missing colons, incorrect indentation, undefined variables

palette

Code Style (PEP 8)

Evaluates adherence to PEP 8 Python style guidelines using Pylint. This includes naming conventions, spacing, and formatting standards.

Example: Variable naming (snake_case), line length, whitespace consistency

settings

Cyclomatic Complexity

Measures code complexity by counting decision points (if, loops, etc.). Lower complexity means code is easier to understand and test.

Range: 1 (simple) to 10+ (complex) | Ideal: ≤ 3

health_and_safety

Maintainability Index

Calculated using lines of code, cyclomatic complexity, and Halstead metrics. Shows how easy it is to maintain and update the code.

Range: 0-100 | 85+ (Excellent), 50-85 (Good), <50 (Poor)

Scoring System (0-100)

Your overall score is calculated by combining three weighted metrics. Each metric is scored independently and then summed to produce the final score.

Metric weights

  • Style Score (PEP 8 Compliance) — 40 pts
    Each style issue deducts points; perfect PEP 8 gives the full 40 points.
  • Complexity Score (Cyclomatic) — 30 pts
    Lower complexity scores best; high complexity reduces the score.
  • Maintainability Score — 30 pts
    Higher maintainability index yields higher points.

Scoring table
